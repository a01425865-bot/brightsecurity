🛡️ Bright Security
Your voice, your city — A lightweight, mobile-first web application for citizens to report and track community issues such as potholes, broken streetlights, garbage accumulation, water leaks, and public safety concerns.

🎯 Overview
Bright Security empowers communities by providing a frictionless way to log infrastructure and safety issues. The app prioritizes clarity, speed, and accessibility — making it easy for citizens of all ages to participate in improving their neighborhoods.
---
✨ Features
Authentication
Simple sign-in / sign-up flow
Demo accounts available for quick testing
User avatar and name displayed in the header
Log out functionality
Report an Issue
Quick-access floating + button
Form fields: title, category, description, location, and optional photo
Simulated GPS location detection
Instant appearance in the main feed after submission
View & Manage Issues
Card-based feed showing all reported issues
Filter by category (Roads, Lighting, Trash, Water, Safety, Other)
Color-coded category badges for quick scanning
Status tracking: Pending → In Progress → Resolved
Mark any issue as resolved with one tap
Design
Mobile-first, responsive layout
Clean card-based UI with warm terracotta and green palette
Smooth animations and toast notifications
Accessible for all ages and device sizes

🚀 Getting Started
Option 1: Open Directly
Simply download or clone the repository and open `index.html` in any modern web browser. No server, build tools, or dependencies required.
```bash
git clone https://github.com/YOUR-USERNAME/bright-security.git
cd bright-security
open index.html
```
Option 2: GitHub Pages
This app is deployed via GitHub Pages and accessible at:
```
https://YOUR-USERNAME.github.io/bright-security/
```

🗂️ Project Structure
```
bright-security/
├── index.html    ← Entire application (HTML + CSS + JS)
└── README.md     ← This file
```
The app is a single-file application — all HTML, CSS, and JavaScript are contained in `index.html` for maximum simplicity and portability.

🛠️ Tech Stack
Technology	Purpose
HTML5	Structure and semantic markup
CSS3 (Flexbox)	Responsive layout and styling
Vanilla JavaScript	App logic and interactivity
Google Fonts	Typography (DM Sans, Playfair Display)
In-Memory Storage	Data persistence during session
No frameworks, no build steps, no external dependencies beyond Google Fonts.
